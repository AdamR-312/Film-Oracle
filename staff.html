Overview
What This Site Is
v1.0  ·  2026

Adam's Cinema is a personal film site built entirely in raw HTML and CSS — no frameworks, no build tools, no dependencies. Every page is a standalone file that runs directly in a browser, which means it can be hosted for free on GitHub Pages and opened on any device without installing anything.

The site was designed and built in collaboration with Claude Sonnet, Anthropic's AI assistant, through an iterative conversation that evolved from a pair of batch files into a full multi-room cinema experience. The aesthetic — velvet, gold, dimmed marquee lights, opening curtains — was a deliberate creative direction chosen to make the site feel like attending a high-end theatre rather than browsing a typical film blog.

Every design decision was made to feel like a place, not a page. Design philosophy

The site is entirely static. There are no servers, no databases, and no backend infrastructure. The one exception is the Movie Oracle, which makes a live call to the Anthropic API when a user submits their questionnaire.

Built With
Technology Stack
HTML5 & CSS3
Every page is a single self-contained HTML file with embedded CSS. No preprocessors, no PostCSS, no build step. What you see in the file is exactly what runs in the browser.
Google Fonts
Three typefaces define the visual identity: Playfair Display for display headings, EB Garamond for body text, and Cinzel for labels and structural elements. Courier Prime is used for code blocks on this page.
Anthropic API
The Movie Oracle makes a direct POST request to Anthropic's /v1/messages endpoint using the claude-sonnet-4 model. The API call happens in the browser via the Fetch API when a user submits the questionnaire.
GitHub Pages
Free static site hosting provided by GitHub. The site is served directly from the repository's main branch with no configuration required beyond enabling Pages in the repository settings.
Letterboxd Export
The Collection page was built from real data exported directly from a Letterboxd account — ratings, curated lists, watch history, and profile information — all parsed and rendered into the page at build time.
Vanilla JavaScript
All interactivity — curtain animations, progress bars, option selection, API calls, and result rendering — is written in plain JavaScript with no libraries or frameworks. No jQuery, no React, nothing to install.
The Oracle
How It Works

The Movie Oracle is a 13-question questionnaire that collects enough signal about a viewer's current mood, taste, and preferences to make a single confident film recommendation. It was designed around the principle that 13 high-signal questions outperform 30 mediocre ones — every question was chosen because it reveals something different about the person that cannot be inferred from the others.

1
User Completes the Questionnaire
The visitor answers 13 questions across four categories: taste anchors (a personal masterpiece and an overrated film), emotional state (weight tolerance, hope at the end), preferences (genre, themes, runtime, subtitles, pace), and open intent (what to avoid, what they want the film to do to them). The submit button remains locked until all 13 are answered.
2
Answers Are Assembled Into a Prompt
JavaScript reads every selected option and text input, maps option values back to their human-readable labels, and constructs a structured natural-language prompt. Scale answers are translated to descriptive phrases — for example, a score of 1 on the emotional weight scale becomes "Something Light" rather than just the number 1.
3
The Prompt Is Sent to Claude
The assembled prompt is sent via a POST request to Anthropic's /v1/messages endpoint using the claude-sonnet-4 model. The Oracle's system instructions tell Claude to behave as a supremely confident film curator, recommend exactly one film with no hedging, and avoid obvious or generic choices in favour of genuine cinematic depth.
4
Claude's Response Is Parsed and Rendered
The response is expected in a structured format — TITLE, DIRECTED BY, WHY, and FIND IT — which is extracted using regular expressions. Each field is then injected into the result card that slides up on screen, styled as a theatre marquee reading Now Showing.
// The Oracle's system persona sent with every request const prompt = `You are Adam's Movie Oracle — a world-class, supremely confident film curator. Recommend exactly ONE film. Be decisive and never hedge. Do not suggest the most obvious choice — dig deep into cinema history to find the perfect match. Format your response EXACTLY as: TITLE: [Film Title] ([Year]) DIRECTED BY: [Director Name] WHY: [3-4 sentences explanation] FIND IT: [One sentence on streaming/rental] Their answers: 1. Personal masterpiece: ${fmt('1')} 2. Overrated film: ${fmt('2')} 3. Emotional weight (1=light, 5=heavy): ${fmt('3')} ...`

The questionnaire was deliberately reduced from an earlier 27-question version after testing showed the additional questions produced minimal improvement in recommendation quality. The 13 questions that survived were kept because each one reveals something the others cannot — the masterpiece and overrated film together triangulate taste far more efficiently than any genre checkbox.

Data
The Collection Page

The Collection page was built entirely from a real Letterboxd data export. Letterboxd provides a complete export of your account under Settings → Data → Export Your Data, which arrives as a zip file containing several CSV files.

Files Used From Export
Seven CSV files were exported from Letterboxd. The following were used to build the Collection page:
ratings.csv profile.csv the-best-movies-i-have-ever-seen.csv fever-dream-films.csv

The CSV data was parsed in Python to extract five-star films, 4.5-star films, curated list positions, profile information, and recent watch history. That data was then hardcoded directly into the HTML — meaning the page requires no live data connection and loads instantly.

To update the collection as new films are watched and rated, simply re-export from Letterboxd, re-parse the new CSV, and replace the relevant sections of collection.html in the GitHub repository.
